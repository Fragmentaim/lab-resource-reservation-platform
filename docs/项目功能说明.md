# 项目功能说明

## 系统定位

实验室资源预约系统面向学校或实验室内部的资源使用场景。普通用户可以查看资源、选择开放时段并提交预约；管理员可以维护资源、配置预约时段、管理用户和查看预约情况。

## 普通用户功能

1. 登录注册
   用户通过用户名和密码登录系统，也可以注册普通用户账号。

2. 资源浏览
   用户可以查看实验室资源列表，按资源名称、资源类型、资源状态等条件筛选。

3. 资源详情
   用户进入资源详情页后，可以查看资源位置、描述和该资源下的可预约时段。

4. 提交预约
   用户选择资源和时段后提交预约。系统会校验资源是否存在、时段是否开放、名额是否充足，以及当前用户是否已经预约过该时段。

5. 我的预约
   用户可以查看自己的预约记录，包括已预约、已取消、已完成等状态。

6. 取消预约
   用户可以取消未开始的预约。取消后预约状态变化，同时释放对应时段的剩余名额。

7. 通知查看
   用户可以查看预约相关通知，并将通知标记为已读。

## 管理员功能

1. 用户管理
   管理员可以查看用户列表，新增用户，编辑用户信息，启用或禁用用户，重置用户密码。

2. 资源管理
   管理员可以维护实验室资源，包括资源编号、资源名称、资源类型、资源状态、位置和描述。

3. 时段管理
   管理员可以给资源配置预约时段，包括起止时间、总名额、剩余名额、时段类型和时段状态。

4. 预约管理
   管理员可以查看所有预约记录，并按用户、资源、状态等条件筛选。

5. 字典管理
   管理员可以维护资源类型、资源状态、时段状态、用户角色等基础枚举数据。

6. 总览看板
   管理员可以查看预约总数、今日预约、资源数量、开放时段和热门资源等统计数据。

## 课堂展示建议

数据库课程展示时，不建议展开 Redis、RocketMQ、异步队列、审计 Outbox 等复杂工程能力。建议重点讲：

- 用户如何预约资源
- 资源和时段为什么是一对多关系
- 预约记录如何连接用户、资源和时段
- 取消预约时如何维护状态和名额
- 字典表如何减少硬编码
- 管理端如何通过数据库查询做分页、筛选和统计
