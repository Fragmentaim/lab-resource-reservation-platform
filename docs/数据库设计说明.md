# 数据库设计说明

## 核心实体

### 用户 sys_user

保存系统用户信息，包括用户名、密码、昵称、角色、手机号和状态。用户分为普通用户和管理员。

主要字段：

- `id`：用户编号，主键
- `username`：用户名
- `password_hash`：密码
- `nickname`：昵称
- `role`：角色
- `phone`：手机号
- `status`：账号状态

### 实验室资源 resource

保存可预约的实验室资源，例如实验室、设备、测试场地等。

主要字段：

- `id`：资源编号，主键
- `resource_code`：资源编码
- `resource_name`：资源名称
- `resource_type`：资源类型
- `status`：资源状态
- `location`：资源位置
- `description`：资源描述

### 预约时段 resource_slot

保存某个资源在某段时间内是否可以预约，以及该时段的预约名额。

主要字段：

- `id`：时段编号，主键
- `resource_id`：资源编号
- `start_datetime`：开始时间
- `end_datetime`：结束时间
- `slot_type`：时段类型
- `total_quota`：总名额
- `remain_quota`：剩余名额
- `status`：时段状态

### 预约记录 reservation

保存用户提交预约后生成的预约记录。

主要字段：

- `id`：预约编号，主键
- `reservation_no`：预约单号
- `user_id`：用户编号
- `resource_id`：资源编号
- `slot_id`：时段编号
- `status`：预约状态
- `cancel_reason`：取消原因
- `created_at`：创建时间

### 用户通知 user_notification

保存系统发给用户的通知消息，例如预约成功、预约取消、提醒通知等。

主要字段：

- `id`：通知编号，主键
- `user_id`：用户编号
- `title`：通知标题
- `content`：通知内容
- `is_read`：是否已读

### 提醒任务 reservation_reminder_task

保存预约开始前需要发送提醒的任务。

主要字段：

- `id`：任务编号，主键
- `reservation_id`：预约编号
- `user_id`：用户编号
- `scheduled_time`：计划发送时间
- `status`：任务状态

### 字典类型 sys_dict_type

保存字典分类，例如资源类型、资源状态、时段状态等。

主要字段：

- `id`：字典类型编号，主键
- `dict_name`：字典名称
- `dict_type`：字典类型编码

### 字典数据 sys_dict_data

保存具体字典项。

主要字段：

- `id`：字典数据编号，主键
- `dict_type`：字典类型编码
- `dict_label`：显示名称
- `dict_value`：实际取值
- `sort_order`：排序

## 实体关系

- 一个用户可以提交多条预约记录：`sys_user 1 : N reservation`
- 一个资源可以拥有多个预约时段：`resource 1 : N resource_slot`
- 一个资源可以被预约多次：`resource 1 : N reservation`
- 一个预约时段可以对应多条预约记录：`resource_slot 1 : N reservation`
- 一个用户可以接收多条通知：`sys_user 1 : N user_notification`
- 一条预约记录可以生成提醒任务：`reservation 1 : N reservation_reminder_task`
- 一个字典类型包含多个字典数据：`sys_dict_type 1 : N sys_dict_data`

## 约束设计

- `sys_user.username` 设置唯一约束，避免用户名重复。
- `resource.resource_code` 设置唯一约束，避免资源编号重复。
- `reservation.reservation_no` 设置唯一约束，保证预约单号唯一。
- `reservation` 中通过 `user_id`、`resource_id`、`slot_id` 关联用户、资源和时段。
- `resource_slot.remain_quota` 记录剩余名额，提交预约时扣减，取消预约时恢复。
